# Vault adapter spec

Every project **engine** (hub, catalog script, agent skill pipeline) must emit notes
that conform to this spec. The vault is a **read-mostly view**; the engine + agent layer
is the source of truth.

## Design principles

1. **Autogenerated by default** — humans do not hand-edit `runs/` or `sources/` notes.
2. **Deterministic IDs** — filename = primary key; sortable and grep-friendly.
3. **YAML frontmatter** — Obsidian Properties + Dataview query surface.
4. **Wikilinks for graph** — causal and topical edges, not just folders.
5. **Domain profiles** — see `docs/00-system/domains/` for per-domain field extensions.

## Record types

| `type` (frontmatter) | Output folder | Description |
|---|---|---|
| `run` | `docs/runs/` | Experiment, backtest, pipeline execution |
| `source` | `docs/sources/{kind}/` | Paper, article, course note, external learning material (see below) |
| `concept` | `docs/concepts/` | Distilled atomic idea (auto-distilled from runs/sources/long-form docs) |
| `moc` | `docs/maps/` | Map of Content dashboard (regenerated, structure stable) |

## Core frontmatter (all types)

```yaml
---
id: "2026-06-11-001-momentum-v3"       # required; matches filename stem
type: run                               # run | source | concept | moc
status: done                            # done | failed | running | archived
date: 2026-06-11                        # ISO 8601 date
tags:                                   # list; see domain profile for conventions
  - run
  - domain/finance
parent_id: "2026-06-10-002-momentum-v2" # optional; predecessor for lineage graph
git_commit: "abc1234"                   # optional; codebase state at run time
source_refs:                            # optional; paths or URLs to L0 artifacts
  - "backtests/2026-06-11/config.yaml"
wikilink_targets:                       # optional; concept note basenames to link
  - "momentum factor decay"
---
```

## Run records (`type: run`)

**Filename:** `{id}.md` where `id` is chronologically sortable, e.g.
`2026-06-11-001-slug` or project-specific flat ids (`gpu__hvg_m2__impact`).

**Required frontmatter extensions:**

```yaml
metrics:                                # domain-specific scalars; flat or nested
  sharpe: 1.42
  max_drawdown: -0.08
hyperparameters: {}                     # optional; tuning config snapshot
dataset_hash: "sha256:…"                # optional; data version fingerprint
engine: "my-hub"                        # which adapter produced this note
regenerate_cmd: "./hub vault"           # how to rebuild this note
```

**Required body sections:**

```markdown
# {title}

**Status** · **Parent:** [[parent-id]] · **Concepts:** [[concept-a]] · [[concept-b]]

## Headline metrics

| Metric | Value |
|---|---|
| … | … |

## What changed

Narrative summary of code/config/data changes (agent-synthesized or engine template).

## On-disk references

Plain paths to L0 artifacts (never embed absolute-path images into synced notes).

---
See also: [[Runs MOC]]
```

## Source records (`type: source`)

**Path:** `docs/sources/{kind}/{id}.md` where `{kind}` matches `source_kind`:

| Subfolder | `source_kind` | Examples |
|-----------|---------------|----------|
| `sources/discussions/` | `discussion` | Cursor chats, podcast distillates |
| `sources/papers/` | `paper` | Academic papers, preprints |
| `sources/articles/` | `article` | Blog posts, news write-ups |
| `sources/courses/` | `course` | Lecture notes, MOOC exports |
| `sources/raw/` | — | L0 PDFs, clips (not markdown notes) |

**Filename:** `{id}.md` e.g. `2026-06-11-001-campbell-ch21` (chronological id + slug).

```yaml
source_kind: paper                      # paper | article | course | book | podcast | discussion
source_path: "sources/raw/campbell.pdf" # L0 path if local
authors: []
year: 2024
```

Wikilinks to sources use the **filename stem only** (e.g. `[[2026-06-11-001-campbell-ch21]]`) so
notes remain link-stable if subfolders change.

**Required body sections:** Summary, Key claims, Methodology (if applicable), Related concepts.

## Concept records (`type: concept`)

Auto-distilled by `/distill-concepts` or engine clustering — not hand-written.

```yaml
source_doc: "framework.md §3.2"         # optional; long-form doc section
distilled_from:                           # optional; run/source ids
  - "2026-06-11-001-momentum-v3"
```

**Body:** One-idea summary, Why it matters, Details, Related (≥2 wikilinks).

## Engine output contract (JSON)

Engines may emit intermediate JSON before markdown render:

```json
{
  "id": "2026-06-11-001-momentum-v3",
  "type": "run",
  "status": "done",
  "date": "2026-06-11",
  "parent_id": "2026-06-10-002-momentum-v2",
  "tags": ["run", "domain/finance"],
  "metrics": {"sharpe": 1.42, "max_drawdown": -0.08},
  "hyperparameters": {"lookback": 20},
  "wikilink_targets": ["momentum factor decay"],
  "body": {
    "title": "Momentum v3 — shorter lookback",
    "what_changed": "Reduced lookback from 60 to 20 days …",
    "on_disk_refs": ["backtests/2026-06-11/config.yaml"]
  }
}
```

A renderer (Python, agent skill, or `jq` + template) converts JSON → markdown file in
the correct folder.

## Validation checklist

Before committing regenerated vault notes:

- [ ] Every `runs/` and `sources/` note has `id`, `type`, `status`, `date`, `tags`
- [ ] Filenames match `id` field
- [ ] No hand-edits in auto folders (or mark `status: archived` if intentionally frozen)
- [ ] `parent_id` resolves to an existing note or is omitted
- [ ] Dataview queries on MOC pages return expected rows
- [ ] No absolute-path image embeds in git-tracked notes
- [ ] Source notes live under `sources/{kind}/` matching `source_kind`
